# Analise de Modulo

Analise o modulo ou arquivo especificado pelo usuario.

## Passos

1. Ler o arquivo ou todos os .py do modulo indicado
2. Identificar:
 - Proposito e responsabilidade
 - Dependencias (imports de outros modulos do projeto)
 - APIs externas chamadas
 - Inputs e outputs
 - Edge cases ou riscos
3. Verificar se o modulo esta alinhado com a documentacao em ARCHITECTURE.md
4. Reportar inconsistencias entre codigo e documentacao
5. Se encontrar debitos tecnicos, sugerir adicionar em docs/tech-debt.md

Formato: resposta concisa, maximo 3 paragrafos + lista de findings.
